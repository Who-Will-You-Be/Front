
## 역할

너는 이 프로젝트의 프론트엔드/백엔드 구조를 정리하고 React 웹 프로토타입을 구현하는 개발 도우미다.

이 프로젝트는 공공데이터 분석 대회용 웹 프로토타입이다.  
완성형 상용 서비스가 아니라, “AI 자기이해 분석 모델이 진로탐색에 어떻게 활용될 수 있는지”를 보여주는 시연용 웹이다.

따라서 로그인, 회원가입, DB 저장, 커뮤니티, 게시판 같은 기능은 만들지 않는다.  
핵심은 다음 흐름이다.

사용자 입력 → 흥미·적성·가치관 분석 → 자기이해 결과 → 탐색해볼 만한 진로 방향 추천

---

## 현재 코드의 문제

현재 코드는 L/R/C/S 네 가지 유형 중 가장 많이 선택된 유형 하나만 뽑아서 결과를 보여준다.

이 방식은 사용하지 마라.

현재 방식의 문제점은 다음과 같다.

1. 흥미·적성·가치관 3가지 분석 축이 반영되지 않는다.
2. Holland RIASEC 기준과 맞지 않는다.
3. 커리어넷 검사 결과를 이미 알고 있는 사용자가 직접 입력할 수 없다.
4. 결과가 단일 유형 중심이라 분석 모델처럼 보이지 않는다.
5. 진로 추천이 너무 단순하다.

따라서 기존 L/R/C/S dominant type 구조를 제거하거나 대폭 수정해라.

## 반드시 구현할 전체 흐름

웹은 다음 페이지 흐름으로 구성한다.

1. MainPage
2. InputModePage
3. DirectInputPage 또는 SituationTestPage
4. ReasonPage
5. ResultPage
6. CareerPage

단순화를 위해 ResultPage와 CareerPage를 합쳐도 된다.  
하지만 사용자가 입력 방식을 선택하는 구조는 반드시 있어야 한다.

## MainPage 요구사항

메인 페이지에는 다음 요소를 넣어라.

- 서비스 제목
- 간단한 설명
- 시작 버튼

제목 예시:

AI 자기이해 기반 진로탐색

설명 예시:

학교생활 속 선택과 이유를 바탕으로  
나의 흥미·적성·가치관을 분석하고  
탐색해볼 만한 진로 방향을 제안합니다.

버튼 문구:

진로 탐색 시작하기

버튼 클릭 시 `/input-mode`로 이동한다.

## InputModePage 요구사항

사용자가 입력 방식을 선택할 수 있는 페이지를 만들어라.

카드 2개를 보여준다.

### 카드 1

제목:

검사 결과 직접 입력

설명:

이미 커리어넷 또는 다른 진로검사 결과를 알고 있다면 직접 입력합니다.

버튼:

직접 입력하기

클릭 시 `/direct-input`으로 이동한다.

### 카드 2

제목:

상황 선택형 검사

설명:

아직 검사 결과를 모르거나 자기이해가 부족하다면 학교생활 상황을 통해 알아봅니다.

버튼:

상황 검사 시작하기

클릭 시 `/situation-test`로 이동한다.

## 분석 축

이 프로젝트는 반드시 3가지 축으로 분석해야 한다.

1. 흥미
2. 적성
3. 가치관

단일 유형만 출력하지 마라.

---

## 흥미 기준

흥미는 Holland RIASEC 기준을 사용한다.

반드시 아래 코드를 기준으로 사용한다.

- R: 현실형
- I: 탐구형
- A: 예술형
- S: 사회형
- E: 기업형
- C: 관습형

주의:

- R은 탐구형이 아니다.
- R은 현실형이다.
- 탐구형은 I이다.
- 창의형은 A에 가깝다.
- 리더십·기획형은 E에 가깝다.

## DirectInputPage 요구사항

사용자가 이미 커리어넷 등에서 검사한 결과를 알고 있을 때 직접 입력하는 페이지다.

이 페이지에서는 흥미, 적성, 가치관 점수를 직접 입력받는다.

입력 방식은 slider 또는 number input을 사용한다.  
점수 범위는 0~100으로 한다.

### 입력 항목

흥미:

- R 현실형
- I 탐구형
- A 예술형
- S 사회형
- E 기업형
- C 관습형

적성:

- 언어능력
- 수리논리력
- 창의력
- 대인관계능력
- 자기관리능력
- 공간지각력
- 손재능
- 예술시각능력

가치관:

- 안정성
- 보수
- 일과삶의균형
- 즐거움
- 자기계발
- 도전성
- 사회적기여
- 자율성
- 성취

입력 완료 버튼을 누르면 `/result`로 이동한다.

이때 state에는 아래 구조로 데이터를 넘긴다.

```js
{
  inputMode: 'direct',
  scores: {
    interest: {
      R: number,
      I: number,
      A: number,
      S: number,
      E: number,
      C: number
    },
    aptitude: {
      언어능력: number,
      수리논리력: number,
      창의력: number,
      대인관계능력: number,
      자기관리능력: number,
      공간지각력: number,
      손재능: number,
      예술시각능력: number
    },
    values: {
      안정성: number,
      보수: number,
      일과삶의균형: number,
      즐거움: number,
      자기계발: number,
      도전성: number,
      사회적기여: number,
      자율성: number,
      성취: number
    }
  }
}
